Linux 提供了 **user**、**group**、**other** 三个纬度，用来控制操作者最资源的访问权限。

这与软件开发中鉴权系统非常相似。Linux 将系统中的**文件**、**目录**、**设备资源**等都抽象成了权限控制对象。通过以上权限级别控制：

- **user**：用户级别权限，表示文件拥有者本人具备的权限
- **group**：组级别权限，Linux 可以将多个用户加入同一组，从而让他们共享一组权限
- **other**：既不是文件拥有者、也不属于文件所属组的其他所有用户

## rwx

Linux 将对文件的和目录的操作划分成了三种基本权限：

- **r**：读权限
- **w**：写权限
- **x**：执行权限

这些权限被组合成了三组，分别对应三个纬度。

```bash
rwxrwxrwx
user | group | other
```

> 对文件来说：rwx 对应读取文件内容，修改文件内容和执行文件
>
> 但对于目录来说：rwx 对应列出目录内容，在目录中创建、删除、重命名文件和进入目录。

我们可以通过以下命令查看文件权限

```bash
ls -l filename
```

可以使用 `chmod` 修改权限、，常见的便是修改一份可执行文件的权限。

```bash
chmod +x filename
```

其中 `+` 和 `-` 都是它的参数，分别代表添加和删减权限。`x` 的位置代表具体操作的权限。

## root

Linux 中存在一个特殊用户 **root**，它拥有系统的最高权限，可以对整个系统执行任何操作。

但是在生产环境中，如果我们使用 **root 用户** 来部署和运行应用（例如 `mysql`、`redis`），一旦某个服务存在漏洞并被攻击者利用，**攻击者可能直接获得 root 权限，从而完全控制系统**。

因此，生产环境应遵循 **最小权限原则**：

- 每个应用程序使用**独立的系统用户**
- 只赋予其运行所必需的最小权限
- 避免服务进程以 root 身份运行

可以通过创建用户、创建组、将用户加入组的方式来进行权限隔离。

相关命令如下：

查看当前用户所属组：

```bash
groups
```

查看指定用户所属组：

```bash
groups username
```

查看系统中所有组：

```bash
cat /etc/group
```

创建组

```bash
groupadd dev
```

删除组

```bash
groupdel dev
```

将用户加入组

```bash
usermod -aG dev username
```

- `-a` 代表追加
- `-G` 指定附加组

从组中移除用户

```bash
gpasswd -d username dev
```

设置用户的主组

```bash
usermod -g dev username
```

修改文件所属组

```bash
chown :dev file.txt
```

创建用户并生成 home 目录

```bash
useradd -m appuser
```

对权限的操作很多都是需要 root 权限的，我们如果是 root 权限，以下切换用户命令可以直接生效，如果不是，则需要输入密码

```bash
su - appuser
```

切换到 root 用户

```bash
sudo su -
```

如果命令前加上 `sudo`，那么它会让命令拥有 root 权限

## user / group / other 权限的判定规则

大家有没有疑问，如果 **user、group、other 三者的权限交叉或重合**会有什么问题。

实际上，在 Linux 中，权限判定是**有严格顺序和覆盖规则的**。

**Linux 的权限判定顺序是**：

```bash
user → group → other
```

当某个用户尝试访问文件或目录时，Linux 内核会按照以下顺序进行判断：

1. **用户权限（user）**
   如果访问者是文件的拥有者（owner），则只使用 user 权限进行检查
2. **组权限（group）**
   如果访问者不是文件拥有者，但属于文件所属组，则只使用 group 权限进行检查
3. **其他权限（other）**
   如果访问者既不是文件拥有者，也不属于文件所属组，则使用 other 权限进行检查

> - **权限不会叠加，一旦匹配到某一类身份，就不会再向下继续判断**
>
> - group 本身并不携带权限，**权限永远是挂在文件上的**。group 的作用是决定**哪些用户可以使用 group 这一列权限**
> - **other 既不属于 owner，也不属于 group 的用户。它规定了访问当前服务器用户最低的权限。**
